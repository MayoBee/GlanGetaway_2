# Image Rendering Issue Fix Plan

## Root Causes (From Analysis)
1. **URL Mismatch**: Database stores `localhost:7002/uploads/` URLs, but API runs on `localhost:5000`; port 7002 server is inactive.
2. **Component Inconsistency**: Edit-Hotel page uses raw `<img>` tags instead of the `SmartImage` component with built-in error handling.
3. **Invalid URL Data**: Database contains literal `"undefined"` strings as image URLs.

## Implementation Steps

### 1. Validate Image URLs Upstream
- **Target Files**: `ManageHotelForm.tsx`, `AccommodationDisplay.tsx`, any other components rendering image lists.
- **Action**: Filter `imageUrls` arrays to exclude invalid entries before passing to components:
  ```typescript
  const validUrls = (urls || []).filter(url => 
    typeof url === 'string' && url.trim() && !['undefined', 'null'].includes(url.trim())
  );
  ```

### 2. Unify Image Components to SmartImage
- **Target File**: `ResortManagementBooking/shared/forms/ManageHotelForm/ManageHotelForm.tsx` (Edit-Hotel page)
- **Action**: Replace all raw `<img>` tags for hotel images with `SmartImage` component from `ResortManagementBooking/hotel-booking-frontend/src/components/SmartImage.tsx` to inherit URL remapping, error handling, and fallback UI.

### 3. Enhance SmartImage Component
- **Target File**: `ResortManagementBooking/hotel-booking-frontend/src/components/SmartImage.tsx`
- **Actions**:
  - Add detection for `ERR_CONNECTION_REFUSED` errors to skip retries for unreachable servers (e.g., port 7002) and immediately show fallback.
  - Verify localhost URL remapping correctly uses the active API server URL (port 5000) instead of hardcoded values.
  - Add fallback invalid URL filtering inside `SmartImage` to handle missed upstream validation.

### 4. Verification
- Run API server on `localhost:5000`.
- Test Edit-Hotel page with port 7002 URLs: confirm no console errors, fallback UI shows for unreachable images.
- Test with `"undefined"` URLs: confirm they are filtered and no broken requests are made.
- Verify consistent image behavior between Detail Page and Edit-Hotel page.
- Confirm no image-related errors in browser console.

## Expected Outcome
- Graceful fallback UI for failed image loads.
- No console errors for image requests.
- Consistent image rendering across all pages.
- Invalid URLs filtered before rendering.
