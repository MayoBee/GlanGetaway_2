# Plan: Fix Resort Display Issue & Full Website Deep Scan

## Issue Overview
Resorts are not appearing on the frontend despite:
- Backend running on port 7002, MongoDB connected successfully
- Frontend running on port 5174
- Database contains 5 hotels all with `isApproved: true`
- API endpoint returns data successfully

Root cause: Data structure mismatch between database schema and frontend component expectations.

## Phase 2: Fix Root Cause Mismatches
### Identified Mismatches:
| Database Field | Frontend Expected Field | Type Difference |
|----------------|-------------------------|-----------------|
| `image`        | `imageUrls`             | string ã array  |
| `rating`       | `starRating`            | same type       |
| `type`         | `type`                  | needs verification |

### Fix Options (Choose One):
**Option A: Fix in Backend (Recommended)**
- Update hotel model to use frontend field names
- OR add field mapping in API endpoint to transform data before sending
- Maintain backward compatibility if needed
- This approach ensures all frontend consumers receive correctly formatted data

**Option B: Fix in Frontend**
- Add data transformation layer in API client
- Map database fields to expected fields when receiving data
- This approach requires no backend changes

**Recommendation: Implement Option A** - Fix in backend API endpoint to return correctly structured data. This is cleaner and avoids requiring every frontend component to handle mapping.
