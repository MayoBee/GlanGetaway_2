# Plan: Fix 409 Conflict on Entrance-Fee Only Booking

## Problem Summary
Booking creation fails with 409 Conflict after successful Stripe payment for entrance-fee only bookings (no rooms/cottages selected). No debug logs from `createAtomicBooking` appear, indicating the request fails before reaching that function.

## Root Cause Analysis
1. **Primary (High Risk)**: Payment intent conflict at `hotels.ts:390-413` — existing booking with same `paymentIntentId` triggers 409 before `createAtomicBooking` is called
2. **Secondary (Medium Risk)**: `createAtomicBooking` data mapping — `selectedItems` array not properly mapped to schema fields `selectedRooms`/`selectedCottages`/`selectedAmenities`
3. **Tertiary (Low Risk)**: Date validation edge case with same-day bookings

---

## Fix Implementation

### Step 1: Tighten Payment Intent Conflict Handling
**File**: `hotels.ts:390-413`

**Add idempotency token to booking payload** (frontend + backend):
- Frontend: Generate UUID idempotency key per booking attempt, send in `BookingFormData`
- Backend: Store `idempotencyKey` in booking document, add index `{userId: 1, idempotencyKey: 1}` with partial filter

**Safe reuse policy** — Update query at `hotels.ts:391`:
```typescript
const existingBooking = await Booking.findOne({ paymentIntentId });
if (existingBooking) {
  const isSameUserAndHotel = 
    existingBooking.userId.toString() === userId && 
    existingBooking.hotelId.toString() === hotelId;
  
  const isReusableStatus = !['confirmed', 'completed'].includes(existingBooking.status);
  
  if (isSameUserAndHotel && isReusableStatus) {
    // Allow retry: return existing booking or update it
    return res.status(200).json({
      message: "Booking already exists for this payment",
      bookingId: existingBooking._id,
      booking: existingBooking
    });
  } else if (!isSameUserAndHotel) {
    // Different user/hotel: block
    console.log("409 Conflict: Payment intent used by different user", {
      existingBookingId: existingBooking._id,
      existingUserId: existingBooking.userId,
      currentUserId: userId,
      existingHotelId: existingBooking.hotelId,
      currentHotelId: hotelId
    });
    return res.status(409).json({ 
      message: "This payment has already been used for a different booking" 
    });
  }
}
```

**Atomic claim** (prevent race conditions):
```typescript
// Use findOneAndUpdate to atomically claim payment intent
const booking = await Booking.findOneAndUpdate(
  { paymentIntentId, status: { $in: ['pending', 'failed'] } },
  { $set: { idempotencyKey: idempotencyKey } },
  { new: true }
);
```

### Step 2: Fix Entrance-Fee Booking Data Mapping
**File**: `availabilityService.ts:122-161` and `hotels.ts:473-508`

**Normalize booking payload in `createAtomicBooking`**:
```typescript
// Map selectedItems to schema fields
const roomItems = selectedItems?.filter(i => i.type === 'room') || [];
const cottageItems = selectedItems?.filter(i => i.type === 'cottage') || [];
const amenityItems = selectedItems?.filter(i => i.type === 'amenity') || [];

const normalizedData = {
  ...bookingData,
  selectedRooms: roomItems.map(({ type, ...rest }) => rest),
  selectedCottages: cottageItems.map(({ type, ...rest }) => rest),
  selectedAmenities: amenityItems.map(({ type, ...rest }) => rest),

// Remove selectedItems if not in schema
delete normalizedData.selectedItems;

// For entrance-fee only: ensure arrays exist
if (!normalizedData.selectedRooms) normalizedData.selectedRooms = [];
if (!normalizedData.selectedCottages) normalizedData.selectedCottages = [];

```

### Step 4: Fix Date Validation
**File**: `hotels.ts:454-465`

Replace zero-hours normalization with business rule:
```typescript
const checkInDate = new Date(checkIn);
const checkOutDate = new Date(checkOut);
const now = new Date();

// Allow same-day bookings: check-in must be >= now (not just date comparison)
if (checkInDate < now) {
  return res.status(400).json({ 
    message: "Check-in date cannot be in the past" 
  });
}

// Business rule: bookings must be at least X hours before check-in (configurable)
const minimumNoticeHours = 0; // Allow immediate same-day bookings
if (checkInDate.getTime() - now.getTime() < minimumNoticeHours * 60 * 60 * 1000) {
  return res.status(400).json({ 
    message: `Bookings must be made at least ${minimumNoticeHours} hours before check-in` 
  });
}
```
