# Plan to Fix Entrance Fee Persistence Issue

## Problem Summary
The entrance fee checkbox in the hotel management form is not persisting correctly. When users check "Include entrance fee" for rooms and save, the frontend sends `enabled: true`, but the database stores `enabled: false`. Consequently, the details page doesn't show the entrance fee indicator, and booking calculations still add entrance fees separately.

## Root Cause Analysis
Based on debug logs:
- Frontend correctly sends FormData with `rooms[0][includedEntranceFee][enabled]: true`
- Backend parses the data correctly and creates updateData with proper includedEntranceFee objects
- Hotel.save() reports success with correct room counts
- However, when fetching the hotel data, all rooms show `includedEntranceFee.enabled: false`

This indicates the save operation is not persisting the includedEntranceFee subdocuments properly, despite markModified() calls.

## Solution Approach
1. **Enhanced Debugging**: Add comprehensive logging before/after save to capture exact data flow
2. **Database Verification**: Directly query MongoDB to confirm stored data
3. **Save Method Optimization**: Test different Mongoose save patterns for embedded subdocuments
4. **Schema Validation**: Ensure Mongoose schema properly handles includedEntranceFee updates
5. **Incremental Testing**: Test with single room updates first, then expand

## Implementation Steps

### Phase 1: Enhanced Debugging & Verification
1. Add pre-save debug logging:
   - Log hotel.rooms before any updates
   - Log sanitizedData.rooms includedEntranceFee values
   - Log hotel.rooms after Object.assign but before save

2. Add post-save debug logging:
   - Log updatedHotel.rooms includedEntranceFee immediately after save
   - Query database directly to verify stored values

3. Deploy and test with debug logs to identify where data is lost

### Phase 2: Save Method Fixes
Based on debug results, implement fixes:
1. Test direct hotel.rooms assignment instead of Object.assign
2. Try hotel.set() with explicit subdocument paths
3. Implement atomic updates for includedEntranceFee fields
4. Add validation middleware to ensure includedEntranceFee persists

### Phase 3: Schema & Data Validation
1. Verify RoomSchema includedEntranceFee definition
2. Check for any pre-save hooks that might reset values
3. Add data validation to ensure includedEntranceFee is preserved
4. Test with different room counts and configurations

### Phase 4: Frontend-Backend Integration Testing
1. Test checkbox save with various combinations
2. Verify details page displays correct indicators
3. Test booking calculations reflect included fees
4. End-to-end testing with real user flows

## Risk Assessment
- **Low Risk**: Debug logging additions won't affect functionality
- **Medium Risk**: Save method changes could affect other hotel fields if not careful
- **High Risk**: Direct database queries in production (mitigate with read-only queries)

## Success Criteria
- Checkbox state persists after save/reload
- Details page shows entrance fee indicator when enabled
- Booking calculations correctly exclude entrance fees when included
- Debug logs show consistent data flow from frontend to database
