# Comprehensive Plan to Fix Authentication Token Storage Issue

## Issue Summary
The authentication system is failing because tokens are not being stored in localStorage after successful login, leading to persistent 401 errors. Despite timing fixes (500ms delay + query invalidation), the core issue persists due to localStorage syntax errors in the `signIn` function.

## Root Cause Analysis
Based on the provided debugging information, the issue stems from syntax errors in localStorage.setItem calls within the signIn function in `api-client.ts`. The backend successfully returns authentication data, but localStorage storage fails, leaving tokens unset and causing authentication state inconsistencies.

### Key Problems Identified:
1. **Syntax Errors in localStorage Calls**: Missing or incorrect quotes in localStorage.setItem calls
2. **Token Storage Failure**: No authentication tokens stored despite successful backend response
3. **Inconsistent Auth State**: React Query validation fails due to empty localStorage, triggering 401 errors
