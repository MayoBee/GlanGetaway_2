## Bug Fix: Admin Login Referer Issue

**Root Cause**: The browser does NOT send `Referer` header on same-origin POST requests. This means the current check `if (referer.includes("/admin-login"))` will ALWAYS fail, even when logging in from /admin-login page.

**Fix Required**:
- Remove the admin origin restriction check entirely from `auth.ts`
- Since admin login page is already completely hidden and only accessible by direct URL, this check is no longer needed
- No security reduction - admin login remains completely hidden from regular users
