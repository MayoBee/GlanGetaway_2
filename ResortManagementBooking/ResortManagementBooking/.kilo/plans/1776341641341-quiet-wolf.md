# Plan: Fix Admin Panel Access Issue

## Root Cause Identified
**The admin panel access is completely blocked at the ProtectedRoute level**:
`packages/shared/auth/ProtectedRoute.tsx:80-83` contains an explicit hardcoded block:
```tsx
// Always redirect if admin access is required (admin UI is disabled)
if (requireAdmin || requireSuperAdmin) {
  return <Navigate to="/" replace />;
}
```

This is why even after successful admin login, any attempt to access `/admin/dashboard` or other admin routes automatically redirects to the homepage.

## Issues Found
1. ✅ **Broken Referer Check Fixed**: Login now works correctly after removing the Referer check
2. ❌ **ProtectedRoute Blocks Admin**: The shared auth ProtectedRoute component actively blocks ALL admin routes
3. ❌ **React Query Mutation Race Condition**: Multiple onSuccess handlers fire on login
4. ❌ **Old Admin Redirect**: There are still references to port 5173 admin portal

## Required Fixes
| File | Changes Needed |
|---|---|
| `packages/shared/auth/ProtectedRoute.tsx` | **Remove the admin block lines 80-83** - this is the main root cause |
| `packages/admin/src/pages/SignIn.tsx` | Use `window.location.href` instead of `navigate()` to bypass race condition |
| `hotel-booking-frontend/src/pages/SignIn.tsx` | Remove the admin redirect logic that sends users to port 5173 |

## Implementation Steps
1. Remove the hardcoded admin block from ProtectedRoute
2. Fix admin signin redirect to use hard page reload
3. Clean up old admin portal redirect references
4. Verify all admin routes are accessible

## Expected Outcome
After fixes:
- Admin logs in successfully at `/admin-login`
- Redirects directly to `/admin/dashboard`
- All admin pages function normally
- No more homepage redirects
