# Implementation Plan: Resort Owner Application UI & Admin Approval Fix

## Issue
1. Users cannot find the `/apply-resort-owner` page - no UI links exist
2. The official Admin Management page (`/admin/admin-management`) does NOT show role promotion requests
3. Only the testing `RolePromotionSimple` page has request viewing/approval functionality

## Implementation Steps

### 1. ✅ Add UI link for users to access application page
**File:** `hotel-booking-frontend/src/components/UsernameMenu.tsx`
- Add "Apply for Resort Owner" link to the user dropdown menu
- Only visible to regular users (not already resort owners/admins)
- Links directly to `/apply-resort-owner`
- Use appropriate icon (Building2 or similar)

### 2. ✅ Fix Admin Management page to show promotion requests
**File:** `packages/admin/src/pages/AdminManagement.tsx`
- Add tab navigation: "Users" / "Role Promotion Requests"
- Implement pending requests view with:
  - User name, email, request date
  - Business permit preview
  - Application description
  - Approve / Decline actions
  - Rejection reason input
- Integrate with existing backend endpoints:
  - GET `/api/role-promotion-requests/pending`
  - POST `/api/role-promotion-requests/:id/approve`
  - POST `/api/role-promotion-requests/:id/decline`

### 3. ✅ Add notification indicator for admins
**File:** `packages/admin/src/pages/Dashboard.tsx`
- Add pending request count badge on the User Management card/link
- Auto-fetch stats from `/api/role-promotion-requests/stats`
- Show pending count as badge indicator

### 4. ✅ Add status indicator for users
**File:** `hotel-booking-frontend/src/components/UsernameMenu.tsx`
- Show "Request Pending" indicator if user has submitted an application
- Disable apply link when request is pending

### 5. ✅ Update ApplyForResortOwner page with proper UI
**File:** `hotel-booking-frontend/src/pages/ApplyForResortOwner.tsx`
- Improve UI to match site design system
- Add proper loading states
- Add success/error toast notifications instead of alerts
- Show request status history

## Backend Status
✅ All backend endpoints are already implemented and working:
- `POST /api/role-promotion-requests` - submit application
- `GET /api/role-promotion-requests` - get user/admin requests
- `GET /api/role-promotion-requests/pending` - admin pending requests
- `GET /api/role-promotion-requests/stats` - admin stats
- `POST /api/role-promotion-requests/:id/approve` - approve request
- `POST /api/role-promotion-requests/:id/decline` - decline request

## Expected Outcome
- Regular users can easily find and submit resort owner applications from their profile menu
- Admins see pending requests directly in the official admin dashboard
- Full approval workflow is integrated properly in the admin UI
- No need for hidden/test URLs anymore
